Step-by-step guide

Publish a Website
with GitHub Pages

From creating a GitHub account to having a live, shareable public URL — for free. No coding experience required.

Step 01 — Create a GitHub Account

Go to github.com/signup and click Sign up. Enter your email, create a password, and choose a username.

  • Go to github.com and click "Sign up"
  • Enter your email address and create a password
  • Choose a username — this will appear in your website URL
  • Verify your email to activate your account
Choose your username carefully.
It becomes part of your public website address:
YOUR_USERNAME.github.io/...
Ready to create your account?

Sign up at github.com

Step 02 — Create a New Repository

A repository is like a folder that stores your website files on GitHub.

  • Go to your GitHub profile and click the Repositories tab
  • Click the green "New" button in the top right

Click "Repositories" tab, then the green "New" button.

  • Enter a Repository name (e.g. my-website)
  • Set visibility to Public — required for free GitHub Pages
  • Click "Create repository"
The repository name appears in your final URL:
username.github.io/repo-name/

Fill in the repo name, set it to Public, and click "Create repository".

Step 03 — Upload Your HTML File

After creating the repository, github.com/new shows a quick setup page. Upload your file from here.

  • Click "uploading an existing file" on the quick setup page

Click "uploading an existing file" to go to the upload screen.

  • Drag and drop your .html file into the upload area, or click "choose your files"
  • Scroll down and click "Commit changes"
Where to find the upload option:
After creating your repo, GitHub shows a "Quick setup" page. Look for the link "uploading an existing file" in the middle of the page and click it — it takes you directly to the drag-and-drop upload screen.

Step 04 — Rename the File to index.html

GitHub Pages serves index.html as the default homepage. Rename your file so it loads automatically when someone visits your URL.

  • Click on your uploaded file in the repository
  • Click the pencil ✏️ icon to open the editor
  • At the top, change the filename to index.html
  • Click "Commit changes…" in the top right
If you skip this step, your site still works but visitors need the full URL ending in /yourfilename.html

Edit the filename field at the top — type "index.html" — then commit.

Step 05 — Enable GitHub Pages

Tell GitHub to serve your repository as a live website.

  • Go to your repository and click the Settings tab in the top navigation bar
  • In the left sidebar, scroll down and click Pages
  • Under "Source", select "Deploy from a branch"
  • Set branch to main and folder to / (root)
  • Click Save

Settings → Pages → select "main" branch → Save.

Step 06 — Your Website is Live

After about 1–2 minutes, go back to Settings → Pages. GitHub will show a green banner with your live URL.

GitHub confirms your site is live with a green banner and clickable link.

Your public website address will be:

🔗 https://YOUR_USERNAME.github.io/YOUR_REPO_NAME/
Anyone can visit this link — no GitHub account needed. It works in any browser, on any device, just like any regular website.
To update your site: go to your repository, click index.html, click the pencil icon, make your edits, and commit. Changes go live automatically within a minute or two.
Quick Summary
  • Create a GitHub account at github.com
  • Create a Public repository
  • Upload your HTML file
  • Rename it to index.html
  • Enable GitHub Pages in Settings
  • Share your live URL with anyone!
