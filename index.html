🌐 Publish Your Website with GitHub Pages

A step-by-step guide to go from an HTML file to a live, shareable public website — for free.

1

Create a GitHub Account

Go to github.com and click Sign up. Enter your email, create a password, and choose a username.

  1. Go to github.com and click "Sign up"
  2. Enter your email address and create a password
  3. Choose a username — this will appear in your website URL
  4. Verify your email to activate the account
💡 Choose your username carefully — it becomes part of your public website address: YOUR_USERNAME.github.io/...
2

Create a New Repository

A repository is like a folder that stores your website files on GitHub.

  1. Go to your GitHub profile and click the Repositories tab
  2. Click the green "New" button in the top right

Click the "Repositories" tab, then the green "New" button to start creating your repository.

  1. Enter a Repository name (e.g. html_website_guide)
  2. Optionally add a description
  3. Set visibility to Public — required for free GitHub Pages
  4. Click "Create repository"
💡 The repository name appears in your final URL: username.github.io/repo-name/
3

Upload Your HTML File

After creating the repository, GitHub shows a quick setup page. Upload your file from here:

  1. Click "uploading an existing file" on the quick setup page

On the quick setup page, click "uploading an existing file" to go to the upload screen.

  1. Drag and drop your .html file into the upload area, or click "choose your files"
  2. Once uploaded, scroll down and click "Commit changes"
4

Rename the File to index.html

GitHub Pages automatically loads a file named index.html as the homepage. Rename your file so visitors don't need to type the filename in the URL.

  1. Click on your uploaded file in the repository
  2. Click the pencil ✏️ icon to open the editor
  3. At the top, change the filename to index.html
  4. Click "Commit changes…" in the top right

Edit the filename field at the top — type "index.html" — then click "Commit changes" to save.

💡 If you skip this step, your site still works but visitors need the full URL ending in /yourfilename.html
5

Enable GitHub Pages

Now tell GitHub to serve your repository as a live website:

  1. Go to your repository main page and click the Settings tab in the top navigation bar
  1. In the left sidebar, scroll down and click Pages
  2. Under "Source", select "Deploy from a branch"
  3. Set the branch to main and folder to / (root)
  4. Click Save

In Settings → Pages, select "main" branch and "/ (root)", then click Save.

6

Your Website is Live! 🎉

After about 1–2 minutes, go back to Settings → Pages. GitHub will show a green banner confirming your site is live.

GitHub confirms your site is live with a green banner and a clickable link.

Your public website address will be:

🔗 https://YOUR_USERNAME.github.io/YOUR_REPO_NAME/
✅ Anyone can visit this link — no GitHub account needed. It works in any browser, on any device, just like a regular website.
✏️ To update your site: go to your repository, click on index.html, click the pencil icon, make your edits, and commit. Changes go live automatically within a minute or two.
